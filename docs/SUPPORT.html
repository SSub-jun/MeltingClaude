Support

Help, FAQ, and troubleshooting for MeltingClaude.

Quick start

1

Install MeltingClaude from the Mac App Store and launch it.

2

On first launch, click Choose ~/.claude/ Folder. macOS will prompt you to confirm read access — this is required because we need to parse your local Claude Code session logs.

3

Pick your subscription plan (Pro / Max 5× / Max 20×) or switch to Custom in Settings to set your own thresholds. Click Connect Claude Code — the menu bar icon updates within seconds.

Frequently asked questions

Why does MeltingClaude need access to my ~/.claude/ folder?

Claude Code stores session logs at ~/.claude/projects/<hash>/<session-id>.jsonl. MeltingClaude reads only those JSONL files to extract token-usage fields (input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens) plus model name and timestamp.

The app is sandboxed — it can only access the folder you explicitly grant via the macOS folder picker. Nothing else on your Mac is touched.

Does MeltingClaude send my data anywhere?

No. The app makes zero network requests — no analytics, no telemetry, no crash reporters, no update checkers. You can verify with Network Link Conditioner or by inspecting it with nettop.

All your usage data is stored locally in a SQLite database inside the app's sandbox container. Uninstalling the app removes it.

The thresholds aren't accurate for my plan. Can I customize them?

Yes. Open Settings (Cmd+, or via the Settings button in the popover) and choose Custom from the plan dropdown. Three text fields appear — set Low / Mid / High caps in tokens. Defaults are seeded from Pro plan estimates.

Anthropic doesn't publish the exact 5-hour rate limits, so the built-in Pro / Max 5× / Max 20× thresholds are best-effort estimates based on community observation.

I use Claude Code on multiple Macs with the same account. Why doesn't it sync?

MeltingClaude reads only the local ~/.claude/ folder on each Mac. There's no built-in account-wide sync.

This means each Mac shows only its own activity. If you use Mac A heavily and switch to Mac B, MeltingClaude on Mac B won't know about Mac A's usage — even though Anthropic's actual rate limit applies account-wide.

Multi-device sync via CloudKit is being explored for a future release.

The icon isn't updating. What's wrong?

Most common causes:

  • Folder not granted: Open the popover. If you see "Reconnect Claude Code" or no data, your folder permission may have been revoked. Use Settings → re-grant access.
  • Claude Code never run: The app needs at least one session in ~/.claude/projects/ to show data. Run Claude Code at least once.
  • Refresh interval too long: Default is 5 seconds. Change in Settings if needed.
How do I uninstall?

Drag MeltingClaude.app to Trash. The sandbox container at ~/Library/Containers/com.byeonjunseob.MeltingClaude/ can be deleted manually if you want to remove all data.

Does it work with Claude Code in IDE extensions (VS Code, Cursor, JetBrains)?

Yes. Any Claude Code session — CLI or IDE — writes to ~/.claude/projects/, and MeltingClaude reads them all.

Does it track Claude.ai chat or the Claude desktop app?

No. Those are separate products from Anthropic and don't write local session logs. MeltingClaude only sees Claude Code (the AI coding tool).

Troubleshooting

"Selected folder doesn't contain a 'projects' subfolder"

You picked the wrong folder. The folder must be your ~/.claude/ directory, which contains a projects/ subfolder created by Claude Code. If ~/.claude/projects/ doesn't exist, run Claude Code at least once first.

Token count seems too high or too low

MeltingClaude counts input + cache_creation + output tokens — matching what counts toward your 5-hour rate limit. Cache reads are not counted (Anthropic policy). If your numbers seem off compared to the Anthropic Console, note that the Console may include different time windows or cache-read tokens.

App icon shows in menu bar but no data appears in popover

This usually means the folder access was granted but no session files were found yet. Either:

  • You haven't used Claude Code since granting access — try a session.
  • Bookmark went stale (rare; happens after major macOS updates) — re-grant in Settings.
